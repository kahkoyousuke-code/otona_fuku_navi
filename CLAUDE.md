# CLAUDE.md — 大人の服選び診断サイト

このファイルは、サイトを作るための設計メモです。
Claude Code はこの内容に沿って実装してください。不明点があれば、勝手に決めず先に質問してください。

---

## 1. このサイトは何か

30〜50代の男性向けの、ファッション診断サイト。
5つの質問に答えると「あなたに似合う服」が分かり、おすすめアイテムを提案する。

### 狙う読者
お金はそこそこ出せるが「何が似合うか分からない」「若作りに見えたくない」「選ぶ時間がない」40〜50代の男性。
一番のニーズは「選ぶ手間」と「買って失敗する不安」をなくすこと。

### このサイトの強み（差別化）
体型を切り口にして「体型別に本当に似合う服」を提案する。
よくあるファッションサイトとの違いはここ。上質感・大人の余裕を全体のトーンにする。

### 収益のしくみ
診断結果のおすすめアイテムにアフィリエイトリンクを置く。
診断で「買う気持ちになった人」を集められるので、通常記事より購入につながりやすい。
ただし押し売り感は出さない。「持っているものはすすめない」方針で信頼を優先する（後述）。

## 2. サイトの構成（必要なページ）

集客から収益までの流れ:
**検索・SNS → 記事 → 診断 → 結果ページ（収益）**

| 種別 | ページ | 役割 |
|---|---|---|
| 入口 | トップページ | 3秒で「自分向けだ」と伝え、診断へ誘導 |
| 主役 | 診断（質問→結果） | 体型・雰囲気から似合う服を提案。収益の中心 |
| 集客 | 記事一覧 / 個別記事 | 検索で読まれる記事。最後に診断へ流す |
| 信頼・法律 | 運営者情報 / お問い合わせ / プライバシーポリシー / 広告表記 | アフィリエイトの審査・SEOに必須 |

最初に作るのは「トップ + 診断 + 結果ページ」。記事と固定ページは後から追加できる構成にする。

## 3. 診断の中身

### 質問（5問・各3択。雰囲気のみ2択）
途中離脱を防ぐため、これ以上は増やさない。

1. **体つきは、どれに近いですか？**（軸・本命）
   - A 肩幅があって、がっしりしている → `gasshiri`
   - B 標準的だと思う → `hyojun`
   - C お腹まわりが気になってきた → `onaka`
2. **どんな雰囲気になりたいですか？**（軸・本命）
   - A きちんと・きれいめに見せたい → `kireime`
   - B リラックスした、こなれた感じ → `casual`
3. **服を着る場面は、どちらが多いですか？**（補助）
   - A 仕事・きちんとした場が多い → `work`
   - B 休日・プライベートが多い → `off`
   - C 半々くらい → `both`
4. **服選びで、一番こまることは？**（補助）
   - A 似合うものが、よく分からない → `fit`
   - B 若作りに見えないか心配 → `young`
   - C 選ぶのが、とにかく面倒 → `tedious`
5. **1着にかけられる予算は？**（補助）
   - A 〜1万円くらい → `low`
   - B 1〜3万円くらい → `mid`
   - C 3万円以上でもよい → `high`

### 振り分けルール（重要）
- **質問1（体型）× 質問2（雰囲気）だけで、行き先の結果ページが決まる。**
  3体型 × 2雰囲気 = 6パターン。
- 残りの質問3・4・5は、行き先は変えず、結果ページの中身を寄せるために使う。
  - 質問3（場面）→ 「仕事の日に / 休日に / オン・オフ兼用で」のラベル表示
  - 質問4（悩み）→ 結果ページの締めの一文を出し分け
  - 質問5（予算）→ おすすめアイテムの価格目安を合わせる

### 6パターンのキー（結果ページとイラストの識別子）
`gasshiri-kireime` / `gasshiri-casual` / `hyojun-kireime` / `hyojun-casual` / `onaka-kireime` / `onaka-casual`

各パターンの結果名・説明文・おすすめ3着の具体的な内容は、
同梱の `fashion-shindan.html` 内の `RESULTS` データをそのまま正とする。

## 4. 結果ページに必要な要素（上から順）

このサイトで唯一お金が生まれる場所。「納得 → 商品 → 次の行動」の順で並べる。

1. **結果の名前**（例「がっしり × きれいめ」）と一言タグライン
2. **完成コーデのイラスト1枚**（固定。詳細は5章）
   「これが、あなたの目指す完成形です」と添える
3. **判定理由**（2〜3行）＋ 質問4の悩みに合わせた締めの一文
4. **おすすめアイテム3着**（収益の心臓部）
   - 各アイテム: 名前 / 似合う理由 / 価格目安（質問5で変動） / 「ここで買える」リンク（アフィリエイト）
   - 各アイテムに **「持ってる」チェック**（5章）
   - アイテムは必ず3〜5着まで。多すぎると選べず買われない
5. **買い足し数の集計**（例「完成まで、あと2着」）
6. **次の行動**（もう一度診断 / このタイプ向けの記事へ）
7. **広告を含む旨の表記**

---

## 5. 仕様で特に注意する2点

### イラスト（固定・1枚）
- 結果ページにつき完成コーデのイラストを **1枚だけ固定表示**。動的に変えない。
- 役割分担: イラスト＝「目指す完成形のお手本」、下のアイテムリスト＝「何を買い足すか（実用）」。
- 画像は AI 生成（水彩・アニメ調、彩度を抑えた落ち着いたトーン、40代の大人の男性、縦長3:4）。
- **ファイル名は診断キーと一致させる**: `images/{キー}.webp`
  例: `images/gasshiri-kireime.webp`
- 画像が未配置でも、仮の枠が表示されて崩れないようにする（現HTMLで実装済み）。
- Web表示用に圧縮（1枚200KB以下目安、WebP推奨）。

### 「持ってる」チェック機能
- おすすめアイテムごとに「持ってる」ボタンを置く。
- 押すと: そのアイテムが薄くなり、価格と「ここで買える」が消え、「すでにお持ちですね」に変わる。再度押すと戻る。
- 残った「持っていないもの」だけにおすすめが集中する。
- 下部の集計に反映（「完成まで、あと◯着」/ 全部所持なら「3着そろっています」）。
- 質問では聞かない（結果が出てから押す方式）。質問を5問に保つため。

## 7. やらないこと / 後回し

- スマホアプリ化はしない（Webで完結。必要ならPWAで代替）
- 最初から6パターン全部を作り込まない。反応の出やすい2〜3パターン
  （標準×きれいめ、がっしり×きれいめ等）から着手し、残りは後追いでよい
- イラストを組み合わせごとに動的生成しない（固定1枚）

## 8. 実装状況（2026-05-31 時点）

Next.js（App Router / Turbopack）+ Vercel 想定で構築済み。`npm run dev`（既定ポート3000）/ `npm run build` で動作確認できる。
このセクションは「今あるもの」の記録。設計意図（1〜7章）はそのまま正とする。

### 完成しているもの
- **診断フロー全体**：トップ `/` → 診断 `/shindan`（5問）→ 結果 `/result/[key]`
  - 質問1×質問2で6パターンのキーに振り分け（`Quiz.tsx`）。質問3・4・5は結果ページの中身（場面ラベル／締めの一文／価格目安）に反映
  - 結果ページに「持ってる」トグル・買い足し集計・イラスト枠（未配置でも崩れない）を実装
- **6パターンすべての結果データ**：`src/data/results.ts`（名前・タグライン・判定理由・おすすめ3着）
- **記事システム**：`content/articles/*.md`（frontmatter: title / description / date / tags）を置くだけで一覧・個別ページ・OGP・診断CTAが自動生成。現在 **41本**
  - Markdown対応記法は `##`/`###`・`**太字**`・`*斜体*`・`- 箇条書き`・段落のみ。**リンク記法 `[]()` と番号リスト `1.` は非対応**（表示が崩れるので使わない）
  - タグ絞り込み（`/articles/tag/[tag]`）と記事末尾の関連記事「あわせて読みたい」を実装。タグの日本語ラベル・グループ分けは `lib/articles.ts` の `TAG_LABELS` / `TAG_GROUPS`
- **法律・信頼ページ**：`/about`（運営者情報）`/contact`（お問い合わせ）`/privacy`（プライバシーポリシー）`/disclosure`（広告表記）。フッターからリンク済み

### フォルダ構成（要点）
- `src/data/questions.ts` … 質問・型・価格/悩み/場面のラベル
- `src/data/results.ts` … 6パターンの結果＋アイテム（`keyword` でZOZO/Yahoo検索、`affiliateUrl` で直リンク上書き）
- `src/data/site.ts` … 運営情報。**公開前に「★要編集」を実値に差し替える**
- `src/lib/articles.ts` … 記事のfrontmatter解析・一覧・タグ・関連記事
- `src/lib/affiliate.ts` … `zozoStoreSearchUrl()` / `itemBuyUrl()`
- `src/components/` … Quiz系・ResultContent・ItemCard・Tally・ArticleList・TagFilter・RelatedArticles・InfoPage・DiagnosisCta

### アフィリエイト方針（現状）
- **Yahoo!ショッピングのZOZOTOWN店主軸**（`store.shopping.yahoo.co.jp/zozo/search.html?p=キーワード` の店内検索にリンク）
- 各アイテムは `keyword` でZOZO店内検索に飛ぶ。ZOZO店内は女性向け商品も含むため keyword に「メンズ」を残す。リンク文言は「ZOZOTOWNで探す」、`rel="...sponsored"` 付き
- **収益化はバリューコマースのLinkSwitch**を `layout.tsx` に1行入れる想定（`store.shopping.yahoo.co.jp` を含む `shopping.yahoo.co.jp` リンクを自動でアフィリエイト化。URL書き換え不要）。審査通過後にタグを差し込む

### 未対応 / 次の候補

- バリューコマース LinkSwitch タグの設置（審査後）
